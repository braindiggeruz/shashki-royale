# Фундаментальный аудит и Roadmap: "Шашки Рояль"

## 1. Анализ текущей ситуации

### 1.1. Проблема с установкой APK
**Симптом:** Ошибка "Приложение не установлено" на Android-устройствах.
**Причина:** APK был переупакован (unzip -> zip) для обновления web-assets, но после этого не был заново подписан. Android (начиная с версии 7.0) требует обязательного наличия валидной подписи (APK Signature Scheme v2/v3). При изменении содержимого архива старая подпись становится недействительной.
**Решение:** Сгенерировать новый debug keystore, выровнять APK с помощью `zipalign` и подписать его с помощью `apksigner`. (Это действие уже выполнено в рамках аудита, APK теперь подписан).

### 1.2. Проблемы с UI/UX (по результатам предыдущих исправлений)
- **Отсутствие кнопки "Назад":** Код был обновлен, компонент `AuthHeader` добавлен на страницы авторизации.
- **Проблемы с отображением на малых экранах:** Формы сделаны scrollable.
- **Локальная игра:** Добавлена возможность играть без регистрации.

### 1.3. Интеграция с Supabase и Google OAuth
- **Supabase:** Базовые настройки (URL, anon key) присутствуют в `.env`.
- **Google OAuth:** Код для обработки callback и сохранения данных профиля написан (`updateProfileWithGoogleData`). Однако, для полноценной работы требуется правильная настройка в Supabase Console (Redirect URLs, Client ID, Client Secret от Google Cloud Console).
